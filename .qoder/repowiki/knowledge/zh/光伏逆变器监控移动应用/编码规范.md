- 业务模块严格遵循 Clean Architecture 目录规范，每个 feature 目录下均包含 data（数据源/仓库实现）、domain（实体/接口/用例）和 presentation（UI/Bloc）子目录。
- UI 状态管理统一采用 BLoC 模式，页面通过 `context.read<T>()` 或 `context.watch<T>()` 与业务逻辑交互，事件命名遵循 `ActionRequested` 格式（如 `AuthCheckRequested`）。
- 服务实例化与依赖绑定集中在 `ServiceLocator.init()` 中完成，外部依赖（Dio, SharedPreferences）优先注册，随后是核心服务、数据源、仓库及 BLoC。
