- **硬件通信层**：采用自定义 UART 帧协议（0xAA帧头+XOR校验+字节转义）连接 ARM 主控与 ESP32 网关，ESP32 负责 MQTT 透传与状态维护。
- **云端交互层**：基于 EMQX Broker，通过 `$share` 共享订阅实现多实例负载均衡，利用 Rule Engine 进行数据清洗并桥接至 Redis Streams。
- **业务逻辑层**：实施“型号模块化”设计，通过 `device_model_field` 和 `protocol` 配置表动态解析不同型号设备的 MQTT 数据，消除硬编码。
- **前端表现层**：Admin 端采用 React Query 管理状态，遵循统一的导航重构方案（如将告警、工单独立分组），并执行严格的 UI 组件标准化（Card、Table、Message API）。
