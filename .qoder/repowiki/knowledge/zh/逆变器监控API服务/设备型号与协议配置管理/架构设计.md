- 采用经典的三层架构：Handler层（`model_handler.go`）负责HTTP请求解析、参数校验及基于角色的访问控制；Service层（`model_service.go`）封装业务逻辑，如型号编码唯一性校验及默认值处理；Repository层（`model_repository.go`）直接通过 `pgxpool` 执行原生SQL，处理与PostgreSQL的交互。
- 数据模型定义在共享的 `internal/model/models.go` 中，包含 `DeviceModel`、`DeviceModelField` 和 `DeviceModelProtocol` 三个核心实体。
- 依赖方向明确：Handler -> Service -> Repository -> Database，各层通过结构体组合进行依赖注入。
