- **启动编排**：`cmd/main.go` 负责初始化 PostgreSQL 和 Redis 连接池，并执行依赖注入（DI），将 Repository 和 Service 实例组装后传递给 Handler。
- **分层架构**：采用标准的 Handler-Service-Repository 三层结构。Handler 处理 HTTP 请求与参数校验；Service 封装核心业务逻辑（如权限校验、状态流转）；Repository 负责数据持久化与缓存交互。
- **跨域共享基础设施**：所有子模块共享 `pkg/jwt`（身份验证）、`pkg/logger`（日志）、`pkg/response`（统一响应格式）以及 `internal/middleware`（全局中间件如 CORS、限流、内部鉴权）。
- **内部通信契约**：通过 `/api/v1/internal` 路由组暴露内部接口，配合 `middleware.InternalAuth` 中间件，实现与设备服务端（Device Server）或其他后端组件的安全通信。
