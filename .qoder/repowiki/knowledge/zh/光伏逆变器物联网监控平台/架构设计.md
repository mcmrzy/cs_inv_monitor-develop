- **统一入口与隔离**：`api-gateway` 作为唯一外部 HTTP 入口，基于路由规则将管理面请求转发至 `inv-api-server`，实现内外网隔离与集中式鉴权。
- **异步数据闭环**：构建“设备 MQTT → Bridge → Kafka → Device Server → TimescaleDB”的遥测链路，并通过 Redis 缓存实现实时状态同步，解耦高频写入与查询。
- **共享基础设施编排**：根目录 `Makefile` 与 `deploy/docker-compose*.yml` 统一定义 PostgreSQL、Kafka、EMQX 等中间件拓扑，确保各微服务在一致的环境中协同运行。
- **标准化质量门禁**：`.github/workflows/ci.yml` 并行执行 Go、TypeScript 及 Flutter 的代码检查与测试，保障跨语言模块集成的稳定性。
