- **核心框架**：基于 `gin-gonic/gin` 构建 HTTP 服务器，采用中间件链式处理架构。
- **分层结构**：`main.go` 负责启动与依赖注入；`internal/routes` 定义路由表与转发规则；`internal/proxy` 封装 `httputil.ReverseProxy` 实现请求转发与路径重写；`internal/middleware` 实现横切关注点（认证、授权、限流、日志）；`internal/config` 管理 YAML 配置与环境变量。
- **依赖方向**：网关依赖 Redis 进行 RBAC 缓存与限流状态存储，依赖 PostgreSQL（通过 `pgx`）作为 RBAC 权限数据的后备存储。
- **接口边界**：对外暴露 `/api/v1/*` 等统一路径，对内根据配置将流量分发至 `APIServer` 或 `DeviceServer`。
