- 所有中间件在处理受保护资源前，均优先调用 `isPublicPath` 白名单检查，对健康检查、登录等公开路径直接放行。
- JWT 认证成功后，统一将 `user_id`, `role`, `phone` 等 Claims 提取并注入到 HTTP 请求头（如 `X-User-ID`），供下游中间件或后端服务使用。
- 限流实现采用令牌桶算法（Token Bucket），同时支持全局维度和基于路由前缀（Path Prefix）的细粒度维度配置。
- 反向代理在 `Director` 阶段统一处理路径规范化，包括移除末尾斜杠以及通过 `RewriteHandler` 实现特定接口的路径映射转换。
