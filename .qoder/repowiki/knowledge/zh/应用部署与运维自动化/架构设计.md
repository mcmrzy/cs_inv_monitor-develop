- **双模部署架构**：支持容器化部署（通过 `docker-compose.yml` 及生产专用 `docker-compose.prod.yml`）与裸机 systemd 部署（通过 `deploy.sh` 动态生成 `.service` 文件）。
- **自动化触发机制**：内置 Python Flask Webhook 服务 (`webhook_server.py`) 处理 Git 事件，以及基于 systemd timer 的 Git Polling 脚本 (`git-poll-deploy.sh`) 实现内网环境下的自动拉取与编译。
- **运维监控体系**：通过 `monitor.sh` 结合 systemd timer 定期检查服务状态、端口连通性及系统资源，并配合 Nginx 反向代理配置与数据库初始化脚本 (`schema.sql`) 完成全链路环境搭建。
