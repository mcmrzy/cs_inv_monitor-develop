- 服务层：`modelApi.ts` 与 `parallelApi.ts` 基于共享 `api` 客户端封装 REST 调用，按资源路径组织 CRUD 与专用操作（如批量更新字段、同步参数、状态与告警查询）。
- 类型契约：`modelApi.ts` 定义强类型接口（型号、字段、协议），`parallelApi.ts` 使用宽松 `any` 以适配动态表单/配置。
- 国际化层：`models.ts` 与 `parallel.ts` 采用 `zh/en` 双语种键值映射，键名按模块前缀隔离（`models.*`、`parallel.*`），覆盖列表、表单、提示与校验文案。
- 依赖方向：服务与文案均不直接耦合 UI 组件，通过统一 API 客户端与标准化键名约定形成清晰边界。
