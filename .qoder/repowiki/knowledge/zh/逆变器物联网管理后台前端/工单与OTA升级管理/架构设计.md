- **分层结构**：采用前后端分离架构，模块内划分为 `services`（数据访问层）与 `locales`（表现层资源）。
- **接口封装**：通过 `workOrderApi.ts` 和 `otaApi.ts` 统一封装 RESTful 请求，依赖底层 `api` 实例处理 HTTP 通信。
- **类型定义**：在 API 文件中直接导出 TypeScript 接口（如 `WorkOrderDetail`），实现数据契约与业务逻辑的强耦合。
- **国际化支持**：使用扁平化的 Key-Value 结构（如 `wo.title`, `ota.firmwareManage`）组织中英文对照，支撑多语言界面渲染。
