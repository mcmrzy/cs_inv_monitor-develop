- **分层架构**：采用标准的 `cmd` (入口), `internal` (业务逻辑), `pkg` (通用工具) 结构。
- **双模数据摄入**：支持直连 MQTT Broker (`internal/mqtt`) 或通过 EMQX Bridge 消费 Kafka 消息 (`internal/service/protocol_parser.go`) 两种模式，由配置动态切换。
- **核心处理流**：`ProtocolParser` 负责从 Kafka 拉取原始报文，依据设备元数据 (`MetadataRepository`) 执行字段映射与规则解析，最终通过内部 HTTP API 转发至主业务系统并更新 Redis 实时缓存。
- **控制通道**：通过 MQTT 客户端订阅命令主题，经由 `DataService` 接收 HTTP 请求后下发控制指令至设备，并处理 OTA 状态回调。
- **依赖方向**：Service 层依赖 Repository 层进行元数据查询，依赖 MQTT/Kafka 组件进行通信，所有外部交互通过 `pkg` 中的封装组件完成。
