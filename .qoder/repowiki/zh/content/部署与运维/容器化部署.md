## 简介
本文件面向容器化部署场景，系统性梳理基于 Docker 与 Docker Compose 的编排方案，覆盖服务定义、依赖关系、网络与存储策略、生产与开发差异化配置、TimescaleDB 定制镜像与优化、以及启动/停止/重启标准流程、日志与监控、故障排查与容器间通信机制等。目标是帮助运维与开发团队在本地与生产环境中稳定、可重复地交付系统。

**更新** 本次更新重点关注中国网络环境的优化改进，通过引入DaoCloud镜像仓库显著提升了镜像拉取速度和构建稳定性，同时完善了健康检查机制和服务依赖管理。

## 项目结构
本项目采用多模块微服务架构，并通过 Docker Compose 将各服务编排为统一的运行单元。核心服务包括：
- 前端管理界面（Nginx 静态托管）
- API 网关（反向代理与路由转发）
- 核心业务 API 服务
- 设备数据接入服务
- MQTT-Kafka 桥接服务
- TimescaleDB（时序数据库）
- Redis 缓存服务
- Prometheus + Grafana（监控告警）
- Mosquitto（MQTT Broker）

## 核心组件
- 前端管理界面：使用 Nginx 托管静态资源，作为用户入口。
- API 网关：统一入口，负责路由转发、CORS、限流、鉴权中间件等。
- 核心业务 API 服务：提供设备、告警、仪表盘、用户等业务接口。
- 设备数据接入服务：解析设备协议、写入 Kafka 或直接入库。
- MQTT-Kafka 桥接服务：桥接 MQTT 主题到 Kafka，供下游消费。
- TimescaleDB：PostgreSQL 扩展，支持时序数据压缩与高性能查询。
- Redis：内存缓存，支持会话存储和键空间事件通知。
- 监控与可视化：Prometheus 抓取指标，Grafana 展示仪表盘。
- Mosquitto：轻量级 MQTT Broker，用于设备上报与下发。

- 服务启动顺序：数据库（TimescaleDB）优先于应用服务；消息中间件（Kafka/Zookeeper）优先于设备接入与桥接服务；前端与网关依赖后端服务可达。

## 健康检查与服务依赖管理

### 健康检查配置详解
各核心服务均配置了专门的健康检查机制：

**数据库健康检查**
- TimescaleDB/PostgreSQL：使用 `pg_isready` 检查数据库连接
- Redis：使用 `redis-cli ping` 检查缓存服务状态
- Kafka：使用 `kafka-topics.sh --list` 验证集群状态
- MQTT-Kafka 桥接：使用 HTTP GET 请求检查 `/health` 端点

**应用服务健康检查**
- API网关服务：HTTP健康检查端点 `/health`，端口8080
- 业务API服务：HTTP健康检查端点 `/health`，端口8080

- 检查间隔：开发环境通常为 5-10 秒，生产环境为 10-30 秒
- 超时时间：5-10 秒，确保快速响应
- 重试次数：3-5 次，避免瞬时故障导致误判
- 启动延迟：30-60 秒，给服务充分的初始化时间

### 服务依赖管理机制
**启动条件类型**
- `service_healthy`：要求依赖服务通过健康检查后才启动
- `service_started`：仅要求依赖服务开始运行即可启动

**依赖层次结构**
```
数据库层 → 缓存层 → 应用服务层 → 网关层 → 前端层
```

- 应用服务依赖数据库和缓存服务（`service_healthy`）
- 网关服务依赖所有应用服务（`service_healthy`）
- 前端服务依赖网关服务（`service_started`）

**更新** 依赖管理已从简单的启动顺序演进为基于健康状态的智能启动机制。

## 故障排查指南
- 服务无法启动：
  - 检查依赖服务健康状态与端口占用。
  - 查看容器日志定位错误原因。
  - 验证健康检查配置是否正确。
- 数据库异常：
  - 确认初始化脚本执行成功。
  - 检查压缩与索引策略是否合理。
  - 验证健康检查命令能否正常执行。
- 消息队列异常：
  - 验证主题创建与权限。
  - 检查消费者组状态与 lag。
  - 确认 Kafka 健康检查配置。
- 监控缺失：
  - 确认指标端点可达与抓取配置正确。
  - 检查防火墙与网络策略。
- 健康检查失败：
  - 检查依赖服务是否真正就绪。
  - 验证健康检查命令的可用性。
  - 调整检查参数（间隔、超时、重试次数）。
- 镜像拉取问题：
  - 检查网络连接和镜像仓库可达性。
  - 验证DaoCloud镜像源配置是否正确。
  - 确认Docker镜像加速器配置。
- 构建速度慢：
  - 检查GOPROXY和npm镜像源配置。
  - 验证阿里云镜像源替换是否生效。
  - 清理Docker构建缓存后重试。

**更新** 新增中国网络环境相关的故障排查方法，包括镜像拉取问题和构建性能问题的诊断步骤。
