## 引言

本指南为基于版本控制的数据库迁移管理提供了全面的实施框架。该系统采用增量迁移策略，通过有序的SQL脚本实现数据库结构的演进，涵盖从初始架构到性能优化再到功能增强的完整生命周期。迁移过程严格遵循安全执行原则，包括自动备份、回滚机制和测试验证流程，确保在不影响生产环境的情况下进行数据库结构变更。

**更新** 本次更新引入了完整的自动迁移执行系统，包含schema_migrations表跟踪机制、基线schema.sql支持和增强的幂等性处理，显著提升了迁移系统的可靠性和可维护性。

系统实现了完整的自动迁移执行引擎，支持版本控制和幂等性保证：

```mermaid
flowchart TD

CHECK_CONFIG --> |禁用| SKIP_MIGRATION[跳过迁移]
SKIP_MIGRATION --> START_SERVER[启动服务]
LOG_COMPLETE --> START_SERVER
```

### schema_migrations表跟踪机制

系统使用schema_migrations表来跟踪所有已应用的迁移版本：

| 字段名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| version | BIGINT | 迁移版本号（从文件名提取） | PRIMARY KEY |
| name | VARCHAR(255) | 迁移文件名 | NOT NULL |
| applied_at | TIMESTAMPTZ | 应用时间戳 | DEFAULT NOW() |

**幂等性保证**：
- 使用`ON CONFLICT DO NOTHING`防止重复插入
- 每次启动前检查版本是否已存在
- 支持部分应用的迁移恢复

### 设备固件版本管理部署流程

**更新** 新增的设备固件版本管理部署流程：

```mermaid
sequenceDiagram
participant Dev as 开发者
participant CI as CI/CD
participant Test as 测试环境
participant Prod as 生产环境
Dev->>CI : 推送固件版本管理代码
CI->>Test : 部署固件版本管理
Test->>Test : 执行版本同步测试
Test->>Test : 验证DSP/BMS版本管理
Test->>CI : 返回测试结果
CI->>Prod : 部署到生产环境
Prod->>Prod : 初始化固件版本管理
Prod->>Prod : 配置版本同步策略
Prod->>Prod : 启动版本监控服务
Prod->>CI : 返回部署状态
CI->>Dev : 通知部署完成
```
