## 功能概述

渠道管理（Channel Management）是管理后台的核心功能模块，为平台超级管理员提供完整的组织、成员、邀请和设备转移管理能力。功能分为 4 个标签页：

| 标签页 | 功能 | 核心能力 |
|--------|------|----------|
| **组织树** | 组织架构管理 | 树形展示、创建/编辑/移动/删除组织 |
| **成员管理** | 组织成员管理 | 添加/移除/角色变更/重新激活 |
| **邀请管理** | 成员邀请 | 发送/撤销邀请、管理邀请状态 |
| **转移审批** | 设备/资源转移 | 审批/拒绝转移请求、批量操作 |

### 技术栈

- **框架：** React 18 + TypeScript
- **UI 库：** Ant Design 5.x
- **数据获取：** TanStack React Query
- **状态管理：** Zustand (authStore)
- **API 客户端：** Axios (channelApi)

---

## 访问控制

**渠道管理功能仅限 `SUPER_ADMIN` 角色使用。**

系统通过 `useAuthStore` 获取当前用户角色进行判断：

```tsx
if (user?.role !== Role.SUPER_ADMIN) {
  // 显示无权限提示
  return <div>无权限访问</div>
}
```

非超级管理员用户访问此页面将看到「无权限」提示卡片，无法查看任何数据。

**权限验证点：**

- 前端：路由级角色检查
- 后端：API 级 RBAC 中间件验证
- 网关：X-Internal-Key 内部通信鉴权

## Tab 1：组织树（Organizations）

### 浏览组织

组织树以树形结构展示整个组织架构，支持多级嵌套：

- **树形展示**：根据 `parent_id` 自动构建层级关系
- **搜索过滤**：顶部输入框支持按名称实时过滤
- **节点展开/折叠**：点击三角图标展开子组织

**组织类型：**

| 类型标识 | 显示名称 | 说明 |
|----------|----------|------|
| `manufacturer` | 制造商 | 设备生产方 |
| `distributor` | 经销商 | 区域分销商 |
| `dealer` | 代理商 | 销售代理 |
| `installer` | 安装商 | 设备安装服务商 |
| `end_user` | 终端用户 | 最终用电客户 |

### 创建组织

点击「新建」按钮打开创建表单：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 组织名称 | 文本输入 | 是 | 组织的显示名称 |
| 组织类型 | 下拉选择 | 是 | 从 5 种类型中选择 |
| 父组织 | 下拉选择 | 否 | 设置上级组织（留空为根组织） |
| 联系人 | 文本输入 | 否 | 组织联系人姓名 |
| 联系电话 | 文本输入 | 否 | 联系电话号码 |
| 地址 | 文本输入 | 否 | 组织地址 |

**操作路径：** `渠道管理 → 组织树 → 点击「新建」按钮 → 填写表单 → 确认`

### 编辑组织

在组织树中点击某个组织节点右侧的「编辑」图标：

- 可修改：名称、类型、联系人、电话、地址
- 不可修改：父组织（需使用「移动」功能）
- 修改后实时生效，无需额外确认

### 移动组织

将组织移动到另一个父组织下（调整组织架构）：

1. 点击目标组织右侧的「移动」图标
2. 在弹窗中选择新的父组织
3. 确认后，该组织及其所有子组织将迁移到新位置

> **注意：** 移动操作会影响所有子组织的层级关系，以及关联数据的组织归属。

### 删除组织

删除组织前需确认：

- 组织下不能有关联的设备（需先转移设备）
- 组织下不能有活跃成员（需先移除成员）
- 删除操作不可恢复

系统会在删除前进行校验，如果条件不满足会提示具体原因。

## Tab 2：成员管理（Members）

**前置条件：** 需在「组织树」标签页中选择一个组织后，成员管理功能才可用。

### 查看成员列表

成员列表展示所选组织的所有成员，包含以下信息：

| 列名 | 说明 |
|------|------|
| 邮箱 | 成员邮箱地址 |
| 手机 | 成员手机号码 |
| 角色 | 成员角色标签（彩色 Tag） |
| 加入时间 | 加入组织的时间（按时区显示） |
| 状态 | 活跃/停用 |
| 操作 | 编辑角色 / 移除 / 重新激活 |

**分页：** 默认每页 20 条，可调整页大小。

### 添加成员

点击「添加成员」按钮打开表单：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 用户邮箱 | 文本输入 | 是 | 需为已注册用户 |
| 角色 | 下拉选择 | 是 | admin / operator / dealer / installer / viewer |

**可用角色：**

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| `admin` | 组织管理员 | 管理成员、设备、配置 |
| `operator` | 运营人员 | 查看和管理设备 |
| `dealer` | 经销商 | 设备销售和基础管理 |
| `installer` | 安装人员 | 设备安装和调试 |
| `viewer` | 查看者 | 仅查看权限 |

### 编辑角色

点击成员行末尾的「编辑」图标：

1. 弹出角色编辑对话框
2. 从下拉列表选择新角色
3. 确认后角色立即生效

> **提示：** 角色变更会影响成员在 App 端的操作权限，建议通知相关用户。

### 移除成员

点击「移除」按钮后弹出确认框：

- 移除后成员将失去对该组织数据的访问权限
- 成员的设备归属不受影响（设备仍属于组织）
- 移除操作可逆（通过「重新激活」恢复）

### 重新激活成员

对于已被移除的成员，操作列会显示「重新激活」按钮：

- 点击后成员恢复活跃状态
- 原有角色和权限自动恢复
- 无需重新邀请

## Tab 3：邀请管理（Invitations）

**前置条件：** 建议在「组织树」中选择组织以过滤显示（可选）。

### 发送邀请

点击「发送邀请」按钮打开表单：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 邮箱地址 | 文本输入 | 是 | 被邀请人邮箱 |
| 目标组织 | 下拉选择 | 是 | 邀请加入的组织（如已选组织则自动填入） |
| 角色 | 下拉选择 | 是 | 受邀人将获得的角色 |
| 有效期 | 数字输入 | 否 | 邀请有效小时数（默认 168 = 7天） |

**邀请流程：**

```
管理员发送邀请
  → 系统生成邀请链接和 Token
  → 发送邀请邮件（含链接）
  → 被邀请人点击链接注册
  → 注册成功后自动加入组织
  → 邀请状态变为「已接受」
```

### 查看邀请列表

邀请列表支持分页和状态过滤：

| 状态 | 颜色标签 | 说明 |
|------|----------|------|
| `pending` | 蓝色 | 等待接受 |
| `accepted` | 绿色 | 已被接受 |
| `revoked` | 灰色 | 已被撤销 |
| `expired` | 红色 | 已过期 |

**列表字段：**

| 列名 | 说明 |
|------|------|
| 邮箱 | 被邀请人邮箱 |
| 角色 | 邀请时指定的角色 |
| 目标组织 | 邀请加入的组织名称 |
| 状态 | 当前邀请状态 |
| 邀请人 | 发送邀请的管理员 |
| 过期时间 | 邀请失效时间 |
| 操作 | 撤销（仅 pending 状态） |

### 撤销邀请

对于状态为 `pending` 的邀请，可点击「撤销」取消：

- 撤销后邀请链接失效
- 被邀请人无法再使用该链接注册
- 撤销操作不可逆（需重新发送新邀请）

## Tab 4：转移审批（Transfers）

### 查看转移请求

转移审批列表展示所有资源转移请求：

| 列名 | 说明 |
|------|------|
| 资源类型 | `device`（设备）或 `station`（电站） |
| 来源组织 | 资源当前所属组织 |
| 目标组织 | 资源要转移到的组织 |
| 请求人 | 发起转移的用户邮箱 |
| 原因 | 转移理由说明 |
| 状态 | pending / approved / rejected |
| 创建时间 | 请求创建时间 |
| 操作 | 批准 / 拒绝 |

**状态过滤：** 可通过状态标签页筛选查看。

### 审批转移

**批准操作：**

1. 点击转移请求行的「批准」按钮
2. 弹出确认对话框
3. 确认后资源立即转移到目标组织
4. 相关成员和设备的访问权限同步更新

**拒绝操作：**

1. 点击「拒绝」按钮
2. 弹出对话框，可填写拒绝原因（可选）
3. 确认后请求人收到拒绝通知
4. 资源保持在原组织不变

### 批量操作

支持通过表格复选框进行批量审批：

1. 勾选多个待审批的转移请求
2. 表格上方出现「批量批准」和「批量拒绝」按钮
3. 批量批准：直接执行，所有选中请求立即生效
4. 批量拒绝：弹出统一拒绝原因输入框

> **效率提示：** 批量操作可大幅提升审批效率，建议集中处理同类请求。

## 常见管理任务

### 任务 1：为新经销商建立组织

1. 打开「组织树」标签页
2. 找到制造商/总代理节点
3. 点击「新建」按钮
4. 填写信息：名称 = "XX 经销商"，类型 = `distributor`，父组织 = 制造商
5. 确认创建
6. 切换到「邀请管理」标签页
7. 发送邀请给经销商负责人

### 任务 2：员工离职处理

1. 切换到「成员管理」标签页
2. 确保已选择目标组织
3. 找到离职员工
4. 点击「移除」按钮确认
5. 如有设备转移需求，在「转移审批」中审批相关请求

### 任务 3：设备跨组织转移

1. 在 App 端由管理员发起转移请求
2. 打开管理后台「转移审批」标签页
3. 找到对应的转移请求
4. 核实来源和目标组织信息
5. 点击「批准」完成转移

### 任务 4：批量审批转移请求

1. 切换到「转移审批」标签页
2. 筛选 `pending` 状态
3. 使用表格复选框选中所有待审批请求
4. 点击「批量批准」
5. 确认操作

### 任务 5：组织架构调整

1. 在「组织树」中使用「移动」功能调整层级
2. 检查受影响的成员和设备
3. 必要时更新成员角色
4. 通知相关用户变更情况

## 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面显示「无权限」 | 当前用户非 SUPER_ADMIN | 联系超级管理员提升角色 |
| 组织树为空 | API 返回空数据 | 检查后端 API 服务是否正常，刷新页面重试 |
| 成员列表无法加载 | 未选择组织 | 先在「组织树」中点击选择一个组织节点 |
| 邀请发送失败 | 邮箱已被邀请或 SMTP 未配置 | 检查该邮箱是否已有 pending 邀请；确认邮件服务配置 |
| 转移审批后无变化 | 缓存未刷新 | 手动刷新页面或等待自动刷新 |
| 删除组织报错 | 组织下有关联资源 | 先转移设备并移除所有成员，再尝试删除 |
| 操作按钮不可见 | 权限不足或状态不匹配 | 确认 SUPER_ADMIN 角色；某些操作仅在特定状态下可用 |
