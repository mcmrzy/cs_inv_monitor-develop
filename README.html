系统架构

                              ┌── JWT 认证（HS256）──┐
                              │   Secret 统一         │
                              ▼                       ▼
  Flutter App ── MQTT (password=JWT) ──→ EMQX Broker ←── 设备(ESP32-C3+ARM) 数据上报
    │                                       │
    │                                 内置 JWT 验签
    │                                 过期自动断连
    │                                 共享订阅分发
    │                                       │
    │                                  ┌────┼────┐
    │                                  ▼    ▼    ▼
    │                           $share/inv-group/   ← EMQX 共享订阅
    │                           inv_device_server-1 / 2 （Go 多实例）
    │                                  │    │
    │                                  ▼    ▼
    │                        PostgreSQL + Redis + Redis Streams
    │
    └── HTTP REST ──→ api-gateway ──→ inv_api_server (Go :8080)
                     (Go :8080)              │
                                             ▼
                                       Admin Frontend (React)
                                       + WebSocket（管理后台推送）

原则：实时走 MQTT 直连 EMQX（低延迟），历史/统计走 HTTP API（按需查询），安全靠 EMQX 内置 JWT 认证。API Gateway 作为统一 HTTP 入口，集中处理认证、权限控制与限流。

子模块详情

1. api-gateway Go 1.25 · Gin

API 网关与反向代理。统一处理 JWT 认证、RBAC 权限控制、限流（令牌桶），将管理面请求反向代理至 inv_api_server。

端口：8080（内部） → 80（外部映射）

2. inv_api_server Go 1.25 · Gin

运维 API 服务。负责用户认证与授权、电站/设备 CRUD、告警管理、WebSocket 实时推送、OTA 固件管理、邮件服务等。

端口：8080

3. inv_device_server Go 1.25 · Gin

设备通信服务。MQTT 共享订阅、设备数据解析、Redis Streams 消费、Kafka 消费、OTA 命令下发、设备在线状态管理。

端口：8081

4. mqtt-kafka-bridge Go 1.25

EMQX Webhook → Kafka 消息桥接服务。将 EMQX 的 Webhook 事件桥接至 Kafka，为高吞吐场景提供异步消息缓冲。

5. inv-admin-frontend React 18 · TypeScript · Vite 5

Web 管理后台。基于 Ant Design v5 + ECharts + Socket.IO + Leaflet，提供设备/用户/电站/告警/型号/OTA 等管理功能。

端口：80（内部） → 3000（外部映射）

6. inv_app Flutter 3.x · Dart ≥3.4.0

跨平台移动端 APP（Android / iOS）。采用 BLoC 状态管理 + go_router 路由 + Dio HTTP 客户端 + mqtt_client 设备直连。

核心特性

⚡ 实时数据

MQTT 直连 EMQX + 共享订阅多实例负载均衡，设备数据毫秒级推送到 APP 端。

📡 OTA 升级

支持 ARM / ESP 双芯片固件远程升级，实时进度跟踪，失败自动标记。

🔐 SN 校验

16 位设备编码 + CRC16-Modbus 校验位，所有入库入口强制校验，无效 SN 自动拒绝。

🏗️ 高可用

EMQX 共享订阅、Redis 共享状态、K8s HPA 自动扩缩 2~10 副本。

📊 数据生命周期

TimescaleDB 自动压缩与连续聚合，配置保留策略自动清理过期数据。

🔒 统一认证

JWT HS256 统一认证：API Server 签发、EMQX 验签，共用同一 Secret。

快速开始

环境要求

  • Flutter 3.x（Dart ≥ 3.4.0）
  • Go 1.25+
  • PostgreSQL 16 + TimescaleDB
  • Redis 7
  • EMQX 5.x
  • Docker & Docker Compose（可选，用于容器化部署）

1. 初始化数据库

# 创建数据库
psql -U postgres -c "CREATE DATABASE inv_mqtt;"

# 导入基础表结构（20+ 表、视图、触发器）
psql -U postgres -d inv_mqtt -f database/schema.sql

# 执行增量迁移（按序号顺序执行）
psql -U postgres -d inv_mqtt -f database/migrations/001_init_schema.up.sql
# ... 依次执行后续迁移文件

# 可选：启用 TimescaleDB 时序优化
psql -U postgres -d inv_mqtt -c "CREATE EXTENSION IF NOT EXISTS timescaledb;"
psql -U postgres -d inv_mqtt -f database/migration_timescaledb.sql

2. 配置 EMQX JWT 认证

在 EMQX Dashboard 中配置 JWT 认证：

字段值
认证方式JWT
JWT 来自于password
加密方式hmac-based
Secret<YOUR_JWT_SECRET>
Secret 使用 Base64 编码不勾选
过期后断开连接✅

3. 启动后端服务

# API Gateway（端口 8080，对外映射 80）
cd api-gateway && go run main.go

# API Server（端口 8080）
cd inv_api_server && go run cmd/main.go

# Device Server（端口 8081）
# MQTT_PASSWORD 需设为有效的 JWT token（由 API Server 签发，Secret 与 EMQX 一致）
export MQTT_PASSWORD="<your_jwt_token>"
cd inv_device_server && go run cmd/main.go

4. 启动 Flutter 应用

cd inv_app
flutter pub get
flutter run

5. Docker Compose 一键启动

cd deploy
docker compose -f docker-compose.yml up -d

这将启动 EMQX、inv_device_server（×2 实例）、inv_api_server、PostgreSQL、Redis 等全部服务。

核心设计

实时数据链路

设备(ESP32+ARM) → EMQX → $share/inv-group/ → inv_device_server(Go)
                                                        ├→ PostgreSQL（时序 + 元数据）
                                                        ├→ Redis（缓存 + Pub/Sub + Streams）
                                                        └→ Kafka（可选，高吞吐缓冲）

Flutter App ← MQTT 直连 EMQX（JWT 认证，订阅 cs_inv/{sn}/#）

App 设备详情页实时数据完全走 MQTT 推送，不通过 HTTP API 轮询。进入页面后 MQTT 订阅即开始推送，下拉刷新不触发额外网络请求。

历史/统计数据链路

App / Web 后台 → HTTP REST → api-gateway → inv_api_server → PostgreSQL 查询返回

电站统计 today_energy 从 device_day_data 实时聚合，确保即使 device_realtime_data 或 station_day_data 数据不完整也能返回正确值。

SN 编号校验

设备 SN 为 16 位编码（例 H1CNA00135000014），所有设备入库入口均强制校验，无效 SN 自动拒绝写入数据库。

校验入口：

  • device_repository.UpsertDeviceInfo（设备信息入库）
  • InternalDeviceStatus（设备状态上报）
  • InternalDeviceData（设备数据上报）

SN 编码结构：

位置含义
0-1制造商（H/O/S + 0-9/A-Z）
2-3国家代码
4-7客户等级 + 数字编号
8-9年月（自定义编码）
10-145 位序列号
15CRC16-Modbus 校验位

高可用设计

  • 共享订阅：$share/inv-group/ 前缀，EMQX 轮询分发消息给多个 inv_device_server 实例
  • Redis 共享状态：多实例通过 Redis HSET 共享设备在线状态
  • Session 清理：SetCleanSession(true) 断开即清理
  • K8s HPA：基于 CPU / 内存自动扩缩 2~10 副本
  • Redis Streams：消费组 + ACK + 死信队列，确保消息不丢失

OTA 固件升级

OTA（Over-The-Air）支持通过管理后台远程升级设备固件，支持 ARM 和 ESP 双芯片独立升级。

升级流程：

管理后台上传固件 → 创建升级任务 → API Server 分发命令
                                      │
                                      ▼
                              inv_device_server（转发 MQTT 命令）
                                      │
                                      ▼
                              cs_inv/{sn}/ota/cmd → 设备
                                      │
                                      ▼
                              设备下载固件 → 升级 → 上报进度
                                      │
                                      ▼
                              cs_inv/{sn}/ota/status → inv_device_server
                                      │
                                      ▼
                              转换格式 → API Server（更新数据库）
                                      │
                                      ▼
                              APP / 管理后台显示进度

MQTT 主题：

主题方向说明
cs_inv/{sn}/ota/cmd下行升级命令（command: start）
cs_inv/{sn}/ota/status上行状态上报（progress / success / failed）

OTA 命令格式：

{
  "command": "start",
  "target": "esp",
  "url": "http://server:8080/firmware/filename.bin",
  "version": "1.0.0",
  "file_size": 1081216,
  "file_md5": "<md5_checksum>",
  "task_id": "73"
}

设备状态上报格式：

{
  "device_id": "H1CNA00135000014",
  "current_version": "1.1.0",
  "state": "upgrading",
  "progress": 45,
  "status_message": "正在下载固件...",
  "error_message": ""
}

功能特性：

  • 支持 ARM / ESP 双芯片独立升级
  • 固件文件自动计算 MD5 和 SHA256
  • 升级失败自动标记任务状态
  • 重复升级自动复用已有任务（避免重复创建）
  • 实时进度跟踪（设备上报 → 数据库 → APP/后台显示）

  • OTA 命令下发（转发升级命令到设备 MQTT 主题）

  • 电站地图展示（Leaflet）
