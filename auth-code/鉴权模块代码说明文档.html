鉴权模块代码说明文档

1. 鉴权架构概述

系统采用 JWT + RBAC 的鉴权架构，API Gateway 作为统一入口进行鉴权，业务服务专注于业务逻辑。

客户端 (App / Web)
API Gateway (:8080)
JWT验证 → RBAC权限检查 → 限流
API Server
用户管理 / Token生成
Device Server
MQTT设备通信

鉴权方式对比

终端类型 协议 认证方式 凭证类型
Web管理端 / App HTTP REST JWT Token 动态凭证 (登录获取)
App (实时数据) MQTT TLS 用户名/密码 用户凭证
设备端 (ESP32) MQTT TLS 固定用户名/密码 固定凭证 (硬编码)

5. 鉴权流程

5.1 登录流程

1. 用户输入账号密码
2. 客户端 POST /auth/login
3. API Server 验证密码 (bcrypt)
4. 生成 Access Token (2h) + Refresh Token (7d)
5. 返回 Token + 用户信息 + 权限列表
6. 客户端存储 Token

5.2 API 请求鉴权流程

1. 客户端请求 Authorization: Bearer {token}
2. API Gateway 验证 JWT 签名
3. 提取 user_id, role 注入请求头
4. RBAC 检查用户角色权限
5. 转发请求到 API Server

5.3 Token 刷新流程

1. API 返回 401 Unauthorized
2. 客户端拦截器捕获
3. POST /auth/refresh (携带 refresh_token)
4. API Server 验证并生成新 Token
5. 客户端更新存储，重试原请求

6. 安全特性

密码安全

  • bcrypt 加密
    密码使用 bcrypt 哈希存储，不可逆
  • 登录失败限制
    5次失败后锁定15分钟
  • 验证码保护
    注册/重置密码需短信/邮箱验证码

Token 安全

  • 双 Token 机制
    Access Token (2h) + Refresh Token (7d)
  • JWT 黑名单
    登出时 Token 加入黑名单
  • HTTPS/TLS
    所有通信加密传输
RBAC 角色说明：
0 - 超级管理员（拥有所有权限）
1 - 原厂    2 - 总代理    3 - 经销商    4 - 安装商    5 - 用户

7. 关键配置

JWT 配置 (config.docker.yaml)

# API Gateway
jwt:
  secret: "${JWT_SECRET}"          # JWT 签名密钥

# API Server
jwt:
  secret: "${JWT_SECRET}"
  expire_time: 2h                 # Access Token 过期时间
  refresh_expire_time: 168h       # Refresh Token 过期时间 (7天)
  issuer: "inv-api-server"         # Token 签发者

RBAC 配置

rbac:
  enabled: true                   # 启用 RBAC
  cache_ttl_sec: 300              # 权限缓存 TTL (5分钟)
