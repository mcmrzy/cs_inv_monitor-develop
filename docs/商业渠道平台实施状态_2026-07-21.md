## 3. 当前工作区已实现但尚未提交

以下改动属于 Task 5，代码已落盘且 Go 回归通过，但仍需评审、真实数据库验证和提交。

### 3.1 JWT 与会话

- Access、Refresh 使用独立 `audience` 和 `token_type`。
- 固定只接受 `HS256`，严格检查 `iss/aud/sub/jti/iat/nbf/exp`。
- Access Token 绑定 `root_tenant_id`、`organization_id`、`membership_id`、成员版本、授权版本、会话版本和 `session_id`。
- Access Token 有效期统一为 15 分钟，Refresh Token 为 7 天。
- Refresh Token 只保存 SHA-256 摘要，不再把原始令牌写入 Redis key。
- Refresh 使用稳定 `session_id` 形成令牌家族；旧令牌重放会原子撤销该家族。
- Redis 黑名单不可用时按拒绝处理，不再 fail-open。
- 密码变更和用户状态变更会提升 `session_version`，使旧令牌立即失效。
- Cookie 已设置 `HttpOnly`、生产环境 `Secure` 和 `SameSite=Strict`。

### 3.2 活动组织上下文

- 新增 `POST /api/v1/auth/context`。
- 使用 Refresh Session 选择活动组织，不接受客户端伪造成员、租户或版本信息。
- 数据库校验用户、成员、当前组织和所有祖先组织状态。
- 组织切换时签发组织绑定 Access Token，并旋转同一 Refresh Token 家族。
- API 所有受保护路由组均接入会话、成员和授权版本即时校验。

### 3.3 网关身份边界

- API 与 Gateway 已统一为 `issuer=inv-api-server`、`audience=inv-platform-api`、token version 2。
- Gateway 先删除客户端提交的 `X-User-*`、租户、组织、成员、会话和授权版本头。
- 只有严格验证后的 Access Token 才能重新注入可信内部身份头。
- Refresh 和 break-glass 类型令牌不能进入普通业务路由。
- RBAC 缺少用户身份或无法解析角色时默认拒绝。
- 日志会脱敏 query 中的 `token/access_token/refresh_token/authorization/code`。

### 3.4 WebSocket

- WebSocket 改为严格解析组织绑定 Access Token。
- 建连时校验黑名单、Refresh Session、会话版本、成员版本和授权版本。
- 设备读取改用活动组织下的 `device:view` 对象级授权，不再只按全局 user ID 判断。
- 连接期间每 30 秒复验过期、撤权和设备授权；失效后主动关闭连接。
- 浏览器 Origin 改为配置白名单，未知 Origin 拒绝。

### 3.5 数据库版本基础

- 新增 067 迁移草案：`users.session_version` 和 `organization_memberships.authorization_version`。
- 用户、成员、角色、权限、资源授权和组织状态变化会提升相应版本。
- 触发器已补充旧主体与新主体同时失效，避免授权转移后旧令牌继续有效。
- 迁移增加已有列规范化、正数约束、默认值和非空约束。

## 6. 当前发布阻断项

以下任一项未关闭时，禁止标记为“可商用”：

1. Gateway 和 API 仍保留全局数字角色模型，`role=0` 的全局绕过尚未彻底移除。
2. 登录响应尚未返回可选 memberships；多组织用户首次登录后的组织选择体验未闭环。
3. `legacy/shadow/enforce/cleanup` 模式和生产启动门禁尚未实现。
4. 组织、成员、邀请、用户转移和配额业务 API 尚未实现。
5. 设备和电站的认领、绑定、分享、解绑、转移状态机尚未实现。
6. 管理端和 Flutter 尚未使用统一权限契约。
7. 067 迁移尚无真实 PostgreSQL 证据，也尚未接入自动集成测试门禁。
8. 迁移编号冲突已在合并前消除：渠道迁移使用 063–067，后续资产迁移从 068 开始。
9. WebSocket 仍接受 query token；虽然日志已脱敏，后续仍应改为一次性 WS ticket 或安全子协议。
10. Linux race、双租户 E2E、容量、灾备、隐私、HIL 和生产凭据验证尚无完整证据。
