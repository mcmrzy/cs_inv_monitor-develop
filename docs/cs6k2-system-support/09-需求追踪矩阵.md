# CS6K2 需求追踪矩阵

## 1. 使用方法

每个合并请求更新受影响行：实现位置、自动测试、HIL/人工证据和状态。状态只允许 `planned / implementing / verified / blocked / accepted-risk`；没有证据不得标为 verified。

## 3. 发布证据目录约定

```text
artifacts/cs6k2-release/<version>/
├─ requirements/       需求基线和变更
├─ ci/                 静态、单元、集成、coverage
├─ contracts/          MQTT/API/schema/DB 对照
├─ hil/                真机配置、版本、测量和结果
├─ performance/        脚本、环境、原始数据和报告
├─ security/           扫描、渗透、凭据轮换（不含秘密值）
├─ deployment/         digest、迁移、备份、回滚和发布单
└─ approvals/          产品/硬件/固件/安全/测试/发布签字
```

自动审查脚本默认输出到 `artifacts/cs6k2-audit/<timestamp>`；该目录应由 CI 作为 artifact 保存，不应提交运行日志到 Git。
