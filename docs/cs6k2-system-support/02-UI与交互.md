# CS6K2 系统支持端 UI 与交互指导

## 1. 设计目标

- 先展示“当前是否安全、设备在做什么”，再提供操作。
- 把终端用户意图、安装参数和厂家定值分层，避免一个万能设置页。
- 所有数值显示单位、数据时间、质量和来源；不把缓存值伪装成实时值。
- 高风险操作显示影响、前置条件、执行阶段和恢复方法。
- 页面由型号 capability 驱动，但布局和安全交互保持一致。

### 2.2 终端用户 App

底部导航建议仅保留：首页、电站/设备、告警、我的。CS6K2 详情内提供：

1. 概览：能源流、SOC、负载、PV、状态和活动告警。
2. 实时：按 AC/电池/PV/温度分组，显示数据时间和质量。
3. 趋势：功率、SOC、能量，按后端粒度查询。
4. 控制：运行、备电 SOC、目标 SOC、能源模式、时间计划。
5. 设备：安装配置只读、固件、网络和服务信息。

## 3. 核心页面要求

### 3.1 设备列表

| ID | 要求 |
|---|---|
| UI-DEV-001 | 支持 SN、型号、状态、电站、安装商、告警、固件和更新时间筛选 |
| UI-DEV-002 | 在线状态至少区分在线、离线、数据过期、接入异常、未知 |
| UI-DEV-003 | 列表只展示最关键的 6–8 列，其余放列设置/详情；移动端卡片化 |
| UI-DEV-004 | 批量操作先做能力交集和权限预检，显示可执行/被排除数量 |
| UI-DEV-005 | 数据为空、加载、失败、部分失败和无权限分别设计，不使用同一个空白页 |

### 3.2 设备概览与能源流

| ID | 要求 |
|---|---|
| UI-OVW-001 | 顶部状态条显示运行模式、在线、最新采样时间、固件和活动故障 |
| UI-OVW-002 | 能源流方向由带符号功率和状态共同决定；异常时用灰色虚线并提示数据不一致 |
| UI-OVW-003 | 关键卡片：PV、负载、电池 SOC/功率、AC 输入、当日能量和温度 |
| UI-OVW-004 | 每个值可查看来源、单位、质量、采样时间和接收时间 |
| UI-OVW-005 | 并机组显示组级数据，避免把主机组数据和成员数据重复相加 |

推荐布局：

```text
┌ 状态条：在线 · 逆变运行 · 采样 3s 前 · 无活动故障 ┐
├ PV ───────→ DC/AC ───────→ 负载                     ┤
│                 ↕                                   │
│               电池 78%       AC输入/旁路             │
├ 今日 PV | 负载 | 充电 | 放电 | 峰值功率             ┤
└ 活动告警 / 配置未同步 / 降额原因                     ┘
```

### 3.3 控制和配置

| ID | 要求 |
|---|---|
| UI-CTL-001 | 使用后端 `control-capabilities` 动态生成分组、表单、范围和禁用原因 |
| UI-CTL-002 | 同屏展示 reported、desired、动态有效范围、最后操作人和同步状态 |
| UI-CTL-003 | 输入控件使用业务单位，前端不处理设备数组下标或 ×10 缩放 |
| UI-CTL-004 | R1 立即确认；R2 二次确认；R3 输入 SN/服务授权并列出前置条件；R4 不在普通页面出现 |
| UI-CTL-005 | 命令卡片展示等待设备、已接收、执行中、已生效、失败、超时和已取消 |
| UI-CTL-006 | 失败展示稳定错误码和用户动作，不只显示“操作失败” |
| UI-CTL-007 | 临时强充/强放必须选择目标、功率和持续时间，并持续显示倒计时 |
| UI-CTL-008 | 离线时可查询历史/配置，但禁止发送非队列型实时命令 |

禁用理由示例：设备离线、BMS 离线、活动故障、温度降额、必须停机、并机从机、无权限、固件版本不足、配置漂移、型号不支持。

### 3.4 并机向导

步骤必须固定：发现成员 → 兼容性检查 → 拓扑配置 → 全组停机确认 → 预配置 → 同步验证 → 启动 → 结果。

| ID | 要求 |
|---|---|
| UI-PAR-001 | 每一步保存草稿但不提前驱动硬件，最终提交前显示完整拓扑 |
| UI-PAR-002 | 成员型号/固件/电池/电站/在线/故障不兼容时明确定位设备 |
| UI-PAR-003 | 三相用 L1/L2/L3 视觉分栏，但不得依靠颜色作为唯一识别 |
| UI-PAR-004 | 失败后显示已执行步骤、回滚结果和当前安全状态 |

### 3.5 告警和工单

| ID | 要求 |
|---|---|
| UI-ALM-001 | 默认突出活动严重告警，历史/已恢复可筛选，不混淆设备状态和人工处理状态 |
| UI-ALM-002 | 详情展示时间线：发生、通知、确认、指派、恢复、处理、关闭 |
| UI-ALM-003 | 遥测快照标出发生前/时/后和缺失原因 |
| UI-ALM-004 | 批量确认/关闭必须限制同类、同权限范围并展示影响数量 |

### 3.6 OTA

| ID | 要求 |
|---|---|
| UI-OTA-001 | 包详情展示 target、型号范围、版本、签名、哈希、大小和发布状态 |
| UI-OTA-002 | 任务创建先预检设备并显示兼容/跳过/阻断原因 |
| UI-OTA-003 | 灰度按批次显示成功率、失败码、暂停原因和实际设备版本 |
| UI-OTA-004 | 不能用单个平均进度掩盖部分失败；支持逐设备下钻 |

## 4. 数据展示规范

- 电压 V：通常 1 位；电流 A：1 位；功率 W：整数或按量级；能量 kWh：2 位；SOC %：1 位；温度 ℃：1 位。
- 小数位由 field capability 提供；禁止页面各自格式化出不同结果。
- 电池电流/功率方向全系统统一，当前协议为正充电、负放电；UI 用文字和箭头辅助。
- `null` 显示“暂无数据”，0 显示 0；不得混用。
- 超过数据新鲜度阈值显示“已过期”，不继续使用绿色在线样式。
- 图表必须显示时区、单位、实际粒度、缺口和质量排除说明。

## 5. 状态、反馈和风险交互

### 5.1 命令状态文案

| 后端状态 | UI 文案 | 颜色/动作 |
|---|---|---|
| pending/queued | 等待发送 | 中性，可取消时显示取消 |
| sent | 等待设备 | 蓝色，不显示成功 |
| acknowledged/executing | 设备执行中 | 蓝色进度 |
| success + reported match | 已生效 | 绿色 |
| success + drifted | 设备已执行，配置未同步 | 黄色，提供重新查询 |
| failed | 执行失败 | 红色，展示拒绝码和建议 |
| timeout | 结果超时，状态待确认 | 黄色，不允许直接重发危险命令 |
| cancelled | 已取消 | 灰色 |

### 5.2 确认要求

- R2：说明供电/电池影响，确认按钮需在内容加载完后可用。
- R3：列出全部前置条件、设备 SN、当前/目标值和回滚方法；必要时输入 SN 后四位。
- 批量 R2/R3：先下载/查看预检报告，不使用一个模糊的“确定”。
- 任何确认都不能替代后端和设备校验。

## 6. 响应式、无障碍和国际化

| ID | 要求 |
|---|---|
| UI-NFR-001 | Web 支持 1280–1920 主工作区；1024 可用；小屏隐藏次要列而非横向无限滚动 |
| UI-NFR-002 | App 适配常用 Android/iOS 尺寸、字体缩放和深色模式 |
| UI-NFR-003 | 交互目标至少约 44×44px，键盘可达，焦点清晰 |
| UI-NFR-004 | 颜色不作为状态唯一载体；图表有文字/图例/表格替代 |
| UI-NFR-005 | 中英文文案使用 locale key；命令 code、错误 code 不翻译 |
| UI-NFR-006 | 日期时间使用电站时区并明确标识；审计同时保留 UTC |

## 7. 前端实现约束

- React Query 管理服务端状态；Zustand 仅保存认证/本地 UI 状态，避免双缓存。
- API DTO 有明确 TypeScript/Dart 类型，禁止页面层直接读取任意 JSON 深层 key。
- App/Web 都从 API 获取 capability；静态设置表仅作兼容回退且默认只读。
- 页面卸载取消订阅/请求；重连后先拉 reported/config revision，再恢复实时订阅。
- 对相同 task_id 的状态更新幂等，旧时间戳/旧 revision 不覆盖新状态。
- 大表格服务端分页；大图表降采样；导出走异步任务。

## 8. UI 验收清单

1. 每页有 loading/empty/error/forbidden/stale/partial 状态。
2. 键盘、读屏、200% 字体和色弱检查通过。
3. 终端用户看不到安装/厂家参数；直接 API 请求同样拒绝。
4. 不支持的能力完全隐藏，支持但暂不可用的能力显示禁用原因。
5. 所有控制显示 task_id 对应的执行阶段，绝不提前成功。
6. 截图基线覆盖桌面、窄屏、中文、英文、浅色、深色和长文案。
