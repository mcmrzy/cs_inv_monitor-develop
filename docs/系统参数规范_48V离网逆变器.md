# CS-I10-6k2 系统参数数据字段规范

**适用产品**: 辰烁科技 CS-I10-6k2 单相 6.2kW 48V 离网逆变器  
**文档版本**: V2.2  
**编写日期**: 2026-06-11  
**配套模块**: BMS（RS485/Modbus）+ MPPT（DC-DC）+ ESP32 WiFi 网关  
**支持并机**: 单相并联扩容 / 三相组网（CAN 总线互联）

---

## 数据格式说明

ESP32 WiFi 模块会为所有 ARM 上报的数据添加 Unix 时间戳，最终上传到 MQTT 云端的数据格式如下：

```json
{
  "data": { /* 原始数据内容 */ },
  "timestamp": 1716800000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `data` | object | ARM 上报的原始数据（各章节定义的 JSON 对象） |
| `timestamp` | int | Unix 时间戳（秒），由 ESP32 NTP 同步提供 |

> **注意**：在线发送和离线缓存发送的数据格式完全统一，均包含时间戳。

| 参数 | Key | 类型 | 单位 | 说明 |
|------|-----|------|------|------|

| ESP 固件 | `firmware_esp` | string | - | WiFi 模块固件版本 |
| DSP 固件 | `firmware_dsp` | string | - | 数字信号处理器固件版本（逆变控制核心） |

## 8. 告警事件

> 主题: `cs_inv/{sn}/data/alarm` | 触发方式: **事件触发**

```json
{
  "code": 8,
  "level": "fault",
  "message": "过温保护",
  "timestamp": 1716800000
}
```

| 参数 | Key | 类型 | 说明 |
|------|-----|------|------|
| 故障码 | `code` | int | 故障编码 |
| 级别 | `level` | string | `warning` / `fault` |
| 描述 | `message` | string | 中文故障描述 |
| 时间戳 | `timestamp` | int | Unix 时间戳 |

**PCS 故障码**：

| 代码 | 名称 | 说明 | 恢复条件 |
|------|------|------|----------|
| 1 | 母线过压 | 直流母线电压超限 | 电压恢复后手动复位 |
| 2 | 母线欠压 | 直流母线电压过低 | 电压恢复正常 |
| 3 | 输出过压 | AC 输出电压 >250V | 电压恢复正常 |
| 4 | 输出欠压 | AC 输出电压 <200V | 电压恢复正常 |
| 5 | 过频 | 输出频率 >50.5Hz | 频率恢复正常 |
| 6 | 欠频 | 输出频率 <49.5Hz | 频率恢复正常 |
| 7 | 孤岛保护 | 检测到孤岛效应 | 电网恢复后自动重连 |
| 8 | 过温保护 | 温度 >85℃ | 温度降至 75℃以下 |
| 9 | PV 过压 | 光伏输入电压 >150V | 电压恢复正常 |
| 10 | 输出过载 | 功率 >6200W | 负载降低后手动复位 |
| 11 | 绝缘阻抗低 | 绝缘阻抗 <500kΩ | 排除漏电后复位 |
| 12 | 输出短路 | AC 输出短路 | 排除短路后复位 |
| 16 | 电池反接 | 电池极性接反 | 纠正接线后复位 |
| 17 | BMS 通信丢失 | 与 BMS 通信中断 | 通信恢复 |

**BMS 故障码**：

| 代码 | 名称 | 说明 |
|------|------|------|
| 0x0101 | 单体过压 | 单体电压超过上限 |
| 0x0102 | 单体欠压 | 单体电压低于下限 |
| 0x0201 | 充电过流 | 充电电流超过限制 |
| 0x0202 | 放电过流 | 放电电流超过限制 |
| 0x0301 | 充电过温 | 充电时温度过高 |
| 0x0302 | 放电过温 | 放电时温度过高 |

## 10. OTA 升级

> 支持 4 个固件目标的远程升级，ESP32 自身通过 HTTP 直接升级，ARM/DSP/BMS 通过 UART 中转升级。

### 10.1 升级路径

```
云端 MQTT 下发 OTA 命令
        │
        ▼
    ┌────────┐
    │ ESP32  │──── 自身 OTA ──→ HTTP 下载 → Flash 写入 → 重启
    └───┬────┘
        │ UART（帧协议）
        ▼
    ┌────────┐
    │  ARM   │──── ARM OTA ──→ IAP 写入 → 校验 → 跳转
    └───┬────┘
        │
   ┌────┴────┐
   ▼         ▼
┌──────┐  ┌──────┐
│ DSP  │  │ BMS  │
└──────┘  └──────┘
  ARM 代为转发固件数据
```

| 升级目标 | target 值 | 路径 | 说明 |
|----------|-----------|------|------|
| ESP32 WiFi 模块 | `esp` | ESP32 自身 | HTTP 下载 + Flash 写入，完成后自动重启 |
| ARM 主控 | `arm` | ESP32 → UART → ARM | ESP32 下载固件后通过 UART 发送给 ARM，ARM 进入 IAP 模式写入 |
| DSP 数字信号处理器 | `dsp` | ESP32 → UART → ARM → DSP | ESP32 下载固件后通过 ARM 中转，ARM 转发给 DSP |
| BMS 电池管理系统 | `bms` | ESP32 → UART → ARM → BMS | ESP32 下载固件后通过 ARM 中转，ARM 通过 RS485 转发给 BMS |

```
→ parallel_set_slave {"id":2}

```
