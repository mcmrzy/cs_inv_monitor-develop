## 1. 交流输出参数

> 主题: `cs_inv/{sn}/data/ac` | 频率: 5s

```json
{
  "voltage": 220.5,
  "current": 8.52,
  "power": 1870.3,
  "apparent": 1875.6,
  "reactive": 120.5,
  "frequency": 50.02,
  "pf": 0.997,
  "load_percent": 62.4,
  "thd_v": 1.8,
  "thd_i": 2.5,
  "dc_injection": 15.0
}
```

| 参数 | Key | 类型 | 单位 | 范围 | 说明 |
|------|-----|------|------|------|------|
| 输出电压 | `voltage` | float | V | 200~250 | 单相 AC 输出 |
| 输出电流 | `current` | float | A | 0~额定 | 实时输出电流 |
| 有功功率 | `power` | float | W | 0~额定 | 实际有功输出 |
| 视在功率 | `apparent` | float | VA | 0~额定 | V × I |
| 无功功率 | `reactive` | float | Var | 0~额定 | - |
| 输出频率 | `frequency` | float | Hz | 49.5~50.5 | 离网模式下由逆变器自身产生 |
| 功率因数 | `pf` | float | - | 0~1 | **必须 ≤ 1** |
| 负载率 | `load_percent` | float | % | 0~100 | 当前负载 / 额定功率 |
| 电压 THD | `thd_v` | float | % | 0~100 | 输出波形质量（离网重点指标） |
| 电流 THD | `thd_i` | float | % | 0~100 | 输出电流谐波 |
| 直流分量 | `dc_injection` | float | mA | - | 输出交流中的直流残留 |

## 2. 电池参数（BMS）

> 主题: `cs_inv/{sn}/data/battery` | 频率: 5s

```json
{
  "soc": 78.5,
  "soh": 96.2,
  "voltage": 51.2,
  "current": 25.5,
  "power": 1305.6,
  "capacity_remain": 78.5,
  "capacity_total": 100.0,
  "cycle_count": 152,
  "temp_max": 28.5,
  "temp_min": 25.0,
  "cell_volt_max": 3.35,
  "cell_volt_min": 3.28,
  "cell_volt_diff": 0.07,
  "charge_state": "charging",
  "battery_type": "LiFePO4",
  "protect_status1": 0,
  "protect_status2": 0
}
```

| 参数 | Key | 类型 | 单位 | BMS 寄存器 | 说明 |
|------|-----|------|------|-----------|------|
| SOC | `soc` | float | % | 0x1000 ÷ 10 | 剩余电量 0~100% |
| SOH | `soh` | float | % | 0x1001 ÷ 10 | 健康状态 |
| 总电压 | `voltage` | float | V | 0x1002 ÷ 10 | 电池组端电压 |
| 电流 | `current` | float | A | 0x1003 ÷ 10 | 正=充电，负=放电 |
| 功率 | `power` | float | W | 计算 | 电压×电流，充放电功率 |
| 剩余容量 | `capacity_remain` | float | Ah | 0x100A ÷ 10 | 当前可用 |
| 额定容量 | `capacity_total` | float | Ah | 0x100B ÷ 10 | 标称值 |
| 循环次数 | `cycle_count` | int | 次 | 0x1009 | 累计充放电循环 |
| 最高温度 | `temp_max` | float | ℃ | 0x1004 ÷ 10 | 电芯最高温度 |
| 最低温度 | `temp_min` | float | ℃ | 0x1005 ÷ 10 | 电芯最低温度 |
| 单体最高电压 | `cell_volt_max` | float | V | 0x1006 ÷ 1000 | - |
| 单体最低电压 | `cell_volt_min` | float | V | 0x1007 ÷ 1000 | - |
| 电芯压差 | `cell_volt_diff` | float | V | 0x1008 ÷ 1000 | 最大值-最小值 |
| 充放电状态 | `charge_state` | string | - | 计算 | `charging` / `discharging` / `idle` |
| 电池类型 | `battery_type` | string | - | 0x1207 | `LiFePO4` / `NCM` / `NCA` / `LeadAcid` |
| 保护状态1 | `protect_status1` | int | - | 0x1020 | 位掩码（过压/欠压/过流/过温/短路等） |
| 保护状态2 | `protect_status2` | int | - | 0x1021 | 位掩码（均衡/通信/绝缘/SOC 等） |

**保护状态1 位定义**（0x1020）：

| 位 | 名称 | 位 | 名称 |
|----|------|----|------|
| Bit0 | 过压保护 | Bit4 | 过温保护 |
| Bit1 | 欠压保护 | Bit5 | 低温保护 |
| Bit2 | 充电过流 | Bit6 | 短路保护 |
| Bit3 | 放电过流 | Bit7 | 反接保护 |

**保护状态2 位定义**（0x1021）：

| 位 | 名称 | 位 | 名称 |
|----|------|----|------|
| Bit0 | 均衡运行中 | Bit3 | SOC 过低 |
| Bit1 | 通信异常 | Bit4 | SOC 过高 |
| Bit2 | 绝缘故障 | Bit5 | 压差异常 |

## 3. 光伏/MPPT 参数

> 主题: `cs_inv/{sn}/data/pv` | 频率: 5s

```json
{
  "pv_voltage": 85.3,
  "pv_current": 12.5,
  "pv_power": 1066.3,
  "mppt_state": "tracking",
  "pv_temp": 52.0
}
```

| 参数 | Key | 类型 | 单位 | 范围 | 说明 |
|------|-----|------|------|------|------|
| PV 电压 | `pv_voltage` | float | V | 0~150 | 48V 系统典型 60~150V |
| PV 电流 | `pv_current` | float | A | 0~额定 | 太阳能板输入电流 |
| PV 功率 | `pv_power` | float | W | 0~额定 | 实时充电功率 |
| MPPT 状态 | `mppt_state` | string | - | - | `tracking` / `standby` / `fault` |
| PV 温度 | `pv_temp` | float | ℃ | -20~85 | 组件或控制器温度（可选） |

> **注意**: 48V 系统中 PV 输入电压通常在 60~150V，与并网逆变器（300~500V）完全不同。

## 4. 系统状态参数

> 主题: `cs_inv/{sn}/data/status` | 频率: 5s

```json
{
  "state": "inverting",
  "fault_code": 0,
  "alarm_code": 0,
  "temp_inv": 48.5,
  "temp_mos": 55.2,
  "temp_ambient": 32.0,
  "dc_bus_voltage": 380.0,
  "fan_speed": 60,
  "efficiency": 94.6
}
```

| 参数 | Key | 类型 | 单位 | 说明 |
|------|-----|------|------|------|
| 运行状态 | `state` | string | - | `standby` / `inverting` / `fault` / `shutdown` |
| 故障代码 | `fault_code` | int | - | 0=正常，非零=具体故障编码 |
| 告警代码 | `alarm_code` | int | - | 0=无告警，非零=具体告警编码 |
| 逆变器温度 | `temp_inv` | float | ℃ | 逆变器内部温度 |
| 功率管温度 | `temp_mos` | float | ℃ | MOS/IGBT 温度 |
| 环境温度 | `temp_ambient` | float | ℃ | 环境温度（可选） |
| 直流母线电压 | `dc_bus_voltage` | float | V | 离网逆变器核心参数 |
| 风扇转速 | `fan_speed` | int | % | 0=停转，1~100=转速百分比 |
| 转换效率 | `efficiency` | float | % | DC→AC 实时效率 |

**状态枚举**：

| state | 说明 |
|-------|------|
| `standby` | 待机，就绪但未输出 |
| `inverting` | 正常逆变输出中 |
| `fault` | 故障，停止输出 |
| `shutdown` | 远程或手动关机 |

## 5. 能量统计参数

> 主题: `cs_inv/{sn}/data/energy` | 频率: 60s

```json
{
  "daily_pv": 8.56,
  "total_pv": 1250.3,
  "daily_charge": 7.80,
  "total_charge": 1100.5,
  "daily_discharge": 6.20,
  "total_discharge": 980.8,
  "daily_load": 6.00,
  "total_load": 950.2,
  "runtime_hours": 8640
}
```

| 参数 | Key | 类型 | 单位 | 说明 |
|------|-----|------|------|------|
| 当日发电量 | `daily_pv` | float | kWh | 当日光伏发电量 |
| 累计发电量 | `total_pv` | float | kWh | 累计光伏发电量 |
| 当日充电量 | `daily_charge` | float | kWh | 当日充入电池（由 ARM 端积分计算） |
| 累计充电量 | `total_charge` | float | kWh | 累计充电量 |
| 当日放电量 | `daily_discharge` | float | kWh | 当日电池放电量 |
| 累计放电量 | `total_discharge` | float | kWh | 累计放电量 |
| 当日用电量 | `daily_load` | float | kWh | 当日负载消耗 |
| 累计用电量 | `total_load` | float | kWh | 累计负载消耗 |
| 运行时长 | `runtime_hours` | int | h | 累计逆变运行小时 |

> **补充**: BMS 端也有充放电累计 Ah（寄存器 0x100C~0x100F），ARM 可将其一并读取后填入 `cs_inv/{sn}/data/cells` 中作为交叉校验。

## 6. 电芯详细数据

> 主题: `cs_inv/{sn}/data/cells` | 频率: 30s

```json
{
  "cell_count": 16,
  "voltages": [
    3.32, 3.33, 3.31, 3.32, 3.35, 3.30, 3.32, 3.31,
    3.33, 3.32, 3.31, 3.32, 3.34, 3.29, 3.32, 3.31
  ],
  "temps": [
    26.5, 26.8, 26.2, 26.5, 27.0, 26.0, 26.5, 26.3,
    26.8, 26.5, 26.2, 26.5, 26.9, 25.8, 26.5, 26.4
  ],
  "charge_ah_total": 12580.3,
  "discharge_ah_total": 12450.6
}
```

| 参数 | Key | 类型 | 单位 | BMS 寄存器 | 说明 |
|------|-----|------|------|-----------|------|
| 电芯数量 | `cell_count` | int | 个 | 0x1208 | 串联电芯数量 |
| 电芯电压数组 | `voltages` | float[] | V | 0x1100~ | 每节电芯电压，系数 0.001V |
| 电芯温度数组 | `temps` | float[] | ℃ | 0x1140~ | 每节电芯温度，系数 0.1℃ |
| 充电累计 Ah | `charge_ah_total` | float | Ah | 0x100C~0x100D | 总充电安时（÷10） |

## 7. 告警事件

> 主题: `cs_inv/{sn}/data/alarm` | 触发方式: **事件触发**（状态变化时立即上报）

```json
{
  "event": "alarm",
  "timestamp": 1715000000,
  "source": "bms",
  "fault_code": 257,
  "fault_desc": "单体过压保护",
  "alarm_code": 0,
  "trigger": {
    "cell_no": 5,
    "cell_voltage": 3.65,
    "threshold": 3.60
  }
}
```

| 参数 | Key | 类型 | 说明 |
|------|-----|------|------|
| 事件类型 | `event` | string | `alarm` / `fault` / `recovery` |
| 时间戳 | `timestamp` | int | Unix 时间戳 |
| 来源模块 | `source` | string | `bms` / `inverter` / `mppt` / `system` |
| 故障码 | `fault_code` | int | 0=正常，非零=具体故障编码 |
| 故障描述 | `fault_desc` | string | 中文故障描述 |
| 告警码 | `alarm_code` | int | 0=无告警，非零=具体告警编码 |
| 触发详情 | `trigger` | object | 触发告警的具体参数（可选） |

**BMS 故障码对照**（详见 `BMS_ARM_Communication_Protocol.md` §7）：

| 代码 | 名称 | 说明 |
|------|------|------|
| 0x0000 | 正常 | 系统正常运行 |
| 0x0101 | 单体过压 | 单体电压超过上限 |
| 0x0102 | 单体欠压 | 单体电压低于下限 |
| 0x0103 | 总压过压 | 总电压超过上限 |
| 0x0104 | 总压欠压 | 总电压低于下限 |
| 0x0201 | 充电过流 | 充电电流超过限制 |
| 0x0202 | 放电过流 | 放电电流超过限制 |
| 0x0203 | 短路保护 | 检测到短路 |
| 0x0301 | 充电过温 | 充电时温度过高 |
| 0x0302 | 放电过温 | 放电时温度过高 |
| 0x0303 | 充电欠温 | 充电时温度过低 |
| 0x0304 | 放电欠温 | 放电时温度过低 |
| 0x0401 | MOS 故障 | 充放电 MOS 异常 |
| 0x0501 | 均衡故障 | 均衡电路异常 |
| 0x0601 | 通信故障 | BMS Modbus 通信中断 |

## 9. 控制参数（云端下发）

> 下行主题: `cs_inv/{sn}/cmd` | 回复主题: `cs_inv/{sn}/cmd/response`

### 9.1 逆变器控制

| 命令 Topic | Payload | 说明 |
|-----------|---------|------|
| `ac_on` | `""` | 交流输出开启 |
| `ac_off` | `""` | 交流输出关闭 |
| `set_voltage` | `{"value":220}` | 设定输出电压 (V) |
| `set_freq` | `{"value":50}` | 设定输出频率 (Hz) |
| `set_power_limit` | `{"value":80}` | 输出功率限制（额定功率百分比 0~100） |
| `eco_mode` | `{"value":1}` | 节能模式：0=关闭, 1=开启 |
| `restart` | `""` | 软重启逆变器 |
| `set_report_interval` | `{"value":5}` | 数据上报间隔 (秒) |
| `query` | `""` | 立即触发一次全量数据上报 |

### 9.2 BMS 控制（ARM 通过 Modbus 写入）

| 命令 Topic | Payload | BMS 寄存器 | 说明 |
|-----------|---------|-----------|------|
| `bms/charge_enable` | `{"value":1}` | 0x1200 | 充放电使能：0=禁止, 1=允许 |
| `bms/charge_current` | `{"value":500}` | 0x1201 | 最大充电电流（×0.1A，500=50A） |
| `bms/discharge_current` | `{"value":1000}` | 0x1202 | 最大放电电流（×0.1A，1000=100A） |
| `bms/charge_volt` | `{"value":584}` | 0x1203 | 充电截止电压（×0.1V，584=58.4V） |
| `bms/discharge_volt` | `{"value":430}` | 0x1204 | 放电截止电压（×0.1V，430=43.0V） |
| `bms/balance_enable` | `{"value":1}` | 0x1205 | 均衡使能：0=禁止, 1=允许 |
| `bms/balance_threshold` | `{"value":50}` | 0x1206 | 均衡启动压差 (mV) |

> BMS 控制值使用 Modbus 原始值（含系数），ARM 端直接写入对应寄存器。

### 9.3 MPPT 控制

| 命令 Topic | Payload | 说明 |
|-----------|---------|------|
| `mppt_on` | `""` | 启用 MPPT 充电 |
| `mppt_off` | `""` | 禁用 MPPT 充电 |
| `mppt_power_limit` | `{"value":1000}` | PV 输入功率限制 (W) |

### 9.4 命令回复格式

```json
{
  "result": "ok",
  "cmd": "bms/charge_current",
  "message": "充电电流已设置为 50.0A",
  "timestamp": 1715000000
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `result` | string | `ok` 成功 / `error` 失败 |
| `cmd` | string | 原始命令 topic |
| `message` | string | 执行结果描述 |
| `timestamp` | int | Unix 时间戳 |

## 10. MQTT 主题总览

### 10.1 上行主题（设备 → 云端）

| 主题 | QoS | 频率 | 说明 |
|------|-----|------|------|
| `cs_inv/{sn}/status` | 1 | 60s | 在线状态 + WiFi RSSI + IP地理位置 |
| `cs_inv/{sn}/info` | 1 | 连接时 | 设备信息（上电/重连后一次） |
| `cs_inv/{sn}/data/ac` | 0 | 5s | 交流输出 + 波形质量 |
| `cs_inv/{sn}/data/battery` | 0 | 5s | 电池综合状态 + 保护状态 |
| `cs_inv/{sn}/data/pv` | 0 | 5s | 光伏/MPPT |
| `cs_inv/{sn}/data/status` | 0 | 5s | 逆变器系统状态 |
| `cs_inv/{sn}/data/energy` | 0 | 60s | 能量统计 |
| `cs_inv/{sn}/data/cells` | 0 | 30s | 电芯详细数据 + BMS 累计 Ah |
| `cs_inv/{sn}/data/alarm` | 1 | 事件 | 告警/故障事件 |
| `cs_inv/{sn}/cmd/response` | 1 | 按需 | 控制命令执行结果 |

### 10.2 status 消息格式（ESP32 自动生成）

> 主题: `cs_inv/{sn}/status` | QoS 1 | Retain true

ESP32 在 WiFi 连接后自动通过 IP 地址获取地理位置，融入 status 消息：

```json
{
  "online": true,
  "rssi": -45,
  "location": {
    "ip": "223.5.5.5",
    "city": "Hangzhou"
  }
}
```

**定位失败时**（如 API 限流或网络问题）：

```json
{
  "online": true,
  "rssi": -45,
  "location": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `online` | bool | 设备在线状态 |
| `rssi` | int | WiFi 信号强度 (dBm) |
| `location` | object/null | IP 定位结果 |
| `location.ip` | string | 公网 IP 地址 |
| `location.city` | string | 城市（英文） |

> **注意**: IP 定位精度为城市级，非 GPS 精确位置。使用 ipapi.co 免费 API，限速 1000 次/天。
> 定位成功后缓存 1 小时，避免频繁请求。

## 11. 数据校验范围

ARM 端上报前需确保数据在合理范围内：

| 参数 | 最小值 | 最大值 | 说明 |
|------|--------|--------|------|
| AC 电压 | 200V | 250V | 单相离网输出 |
| AC 电流 | 0A | 额定电流 | 视逆变器功率 |
| AC 功率 | 0W | 额定功率 | 不超过标称值 |
| 输出频率 | 49.5Hz | 50.5Hz | 离网自生成频率 |
| 功率因数 | 0.00 | 1.00 | **严禁超过 1.0** |
| PV 电压 | 0V | 150V | 48V 系统 MPPT 范围 |
| PV 电流 | 0A | 30A | 视 MPPT 规格 |
| 电池电压 | 40V | 60V | 48V 16S LiFePO4 典型范围 |
| SOC | 0% | 100% | - |
| 温度（逆变器） | -20℃ | 85℃ | 工作温度范围 |
| DC 母线电压 | 0V | 450V | 视电路设计 |

## 12. 分阶段实施建议
